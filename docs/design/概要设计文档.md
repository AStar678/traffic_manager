## 四、系统概述

### 4.1 项目背景

本项目面向智慧交通与智能座舱场景，构建一个前后端分离的Web系统，融合**计算机视觉**（车牌检测OCR、人体关键点检测、手部关键点检测）、**大语言模型**（LLM API调用、自然语言告警摘要生成）和**智能体技术**（自主感知-决策-告警），完成从算法推理到Web可视化的完整链路。

### 4.2 系统功能范围

| 模块 | 功能描述 | 优先级 | 分值 |
|---|---|---|---|
| 道路车辆车牌识别 | 图片上传识别、视频流输入识别、历史记录存储与查询、Web端可视化展示 | P0 | 20 |
| 交警手势识别 | 8种中国标准交警手势识别、人体关键点检测+时序分类、Web端骨架可视化 | P0 | 20 |
| 车主手势控车 | 6种预定义手势识别、手部关键点检测、模拟车辆控制面板、防误触发机制 | P0 | 20 |
| 告警智能体 | 4类监控日志、Agent感知-决策-执行架构、LLM告警摘要生成、多渠道推送、告警仪表盘 | P0 | 15 |
| 用户管理 | 账号密码登录、JWT认证鉴权 | P0 | — |
| 扩展功能 | 多样化登录、Swagger文档、Git规范管理、新模型引入、数据加密 | P1-P2 | 15（可选） |

### 4.3 设计目标

| 目标维度 | 描述 |
|---|---|
| **模块化** | 前后端分离，后端分层（Controller-Service-Repository），算法服务独立部署 |
| **可扩展** | 统一推理接口支持通过 `task_type` 参数扩展新识别任务类型 |
| **实时性** | WebSocket推送告警和识别结果，单张图片识别 ≤ 5秒，手势识别 ≤ 500ms |
| **可靠性** | 异步任务支持失败重试，文件存储持久化，数据库事务保障 |
| **安全性** | JWT认证鉴权、bcrypt密码加密、文件上传白名单校验、CORS跨域控制 |

|---|---|---|
| **前端表现层** | 页面渲染、用户交互、文件上传、WebSocket消息接收、ECharts图表展示 | 不直接访问算法微服务、不直接操作数据库 |

#### 6.1.4 前端核心交互流程

**图片推理流程：**

```
用户选择图片 → FileUploader上传文件 → Axios POST /api/v1/inference/image
→ 后端保存文件 + 调用算法服务 → 返回结构化结果
→ ResultPanel展示结果 + ImageCanvas/KeypointCanvas叠加标注
→ 结果存入ResultPanel本地缓存（最近10条）
```

**异步视频任务流程：**

```
用户选择视频 → FileUploader上传 → Axios POST /api/v1/jobs 创建任务
→ 前端轮询 GET /api/v1/jobs/{job_id}（2秒间隔）
→ VideoPlayer播放视频 + DetectionOverlay逐帧叠加检测结果
→ 任务完成/失败时轮询停止 + 状态提示
```

**WebSocket告警推送流程：**

```
AlertStore.connectWebSocket() → new WebSocket('ws://host/ws/alerts')
→ onmessage: 解析JSON → AlertStore.alerts.unshift(alert) + unreadCount++
→ NavBar铃铛图标红点提示 + AlertTimeline实时追加新条目
```

#### 6.2.2 各Controller详细设计

**AuthController — 用户认证**

| 接口 | 方法 | 路径 | 认证 | 请求参数 | 返回 |
|---|---|---|---|---|---|
| 注册 | POST | `/api/v1/auth/register` | 否 | `{username, password, email}` | `{token, user}` |
| 登录 | POST | `/api/v1/auth/login` | 否 | `{username, password}` | `{token, refreshToken, user}` |

**InferenceController — 统一图片推理**

| 接口 | 方法 | 路径 | 认证 | 请求参数 | 返回 |
|---|---|---|---|---|---|
| 图片推理 | POST | `/api/v1/inference/image` | 是 | `{task_type, image_url}` | `{request_id, results[], annotated_image_url}` |

`task_type` 枚举值：`license_plate`、`police_gesture`、`owner_gesture`

**处理流程：**

```
1. JWT认证校验（SecurityConfig拦截器）
2. 参数校验（task_type + image_url非空）
3. 调用FileStorageService确保文件可用
4. 通过AlgorithmClient转发请求至算法服务 POST /api/v1/inference/image
5. 收到算法服务结构化结果JSON
6. 将结果存入DetectionResult/InferenceRecord表
7. 组装统一响应格式返回前端
```

#### 6.3.2 算法Pipeline设计

**车牌识别Pipeline（LicensePlatePipeline）：**

```
输入: image_url
  ↓
1. image_utils.download_image(image_url) — 下载图片到本地
  ↓
2. detector.detect(image) → 车牌检测框列表 [{bbox, confidence}]
  ↓
3. 对每个检测框:
   ocr.recognize(plate_crop) → 车牌号 + OCR置信度
   color_classifier.classify(plate_crop) → 颜色 (blue/green/white)
  ↓
4. 绘制标注图（绿色检测框 + 车牌号标签）
  ↓
输出: {
  "results": [{
    "plate_number": "京A·12345",
    "plate_color": "blue",
    "bbox": {"x1", "y1", "x2", "y2"},
    "center": {"x", "y"},
    "detection_confidence": 0.95,
    "ocr_confidence": 0.92,
    "overall_confidence": 0.87
  }],
  "annotated_image_url": "...",
  "vehicle_count": 3
}
```

**交警手势Pipeline（PoliceGesturePipeline）：**

```
输入: image_url (单帧) 或 video_url (多帧)
  ↓
1. image_utils 下载图片/视频帧
  ↓
2. pose_detector.detect(image) → 人体33个关键点 [{name, x, y, score}]
  ↓
3. 单帧: gesture_classifier.classify_static(keypoints) → 手势分类
   多帧: 累积N帧关键点序列 → ST-GCN/LSTM推理 → 时序动作分类
  ↓
4. 绘制人体骨架标注图
  ↓
输出: {
  "gesture_code": "STOP",
  "gesture_name": "停止信号",
  "confidence": 0.89,
  "keypoints": [
    {"name": "left_shoulder", "x": 120, "y": 200, "score": 0.95},
    {"name": "left_wrist", "x": 150, "y": 180, "score": 0.91},
    ...
  ],
  "annotated_image_url": "...",
  "frame_index": 0
}
```

**车主手势Pipeline（OwnerGesturePipeline）：**

```
输入: image_url 或 camera_frame
  ↓
1. hand_detector.detect(image) → 手部21个关键点 [{name, x, y, score}]
  ↓
2. gesture_classifier.classify(keypoints) → 手势编码
   静态: 几何规则（手指伸展角度判断）
   动态: 后端连续发送多帧请求，算法侧累积N帧坐标序列判定
  ↓
3. 防误触发: confidence ≥ 0.7 + 持续时间 ≥ 0.5秒
  ↓
输出: {
  "gesture_code": "003",
  "gesture_name": "单指画圈",
  "action": "调节音量",
  "confidence": 0.88,
  "keypoints": [
    {"name": "wrist", "x": 200, "y": 300, "score": 0.97},
    {"name": "index_finger_tip", "x": 210, "y": 280, "score": 0.93},
    ...
  ],
  "is_triggered": true
}
```

#### 6.3.3 视频异步任务处理流程

```
POST /api/v1/jobs (task_type, video_url, callback_url, options)
  ↓
1. 创建任务记录（job_id, status="queued"）
2. 加入任务队列（asyncio.Queue）
  ↓
3. Worker从队列取出任务 → status="processing"
  ↓
4. VideoCapture打开视频 → 按options.fps抽帧
  ↓
5. 逐帧处理循环:
   for each frame:
     pipeline.process(frame) → result
     http_client.callback_result(callback_url, {
       "job_id": "...",
       "event": "frame_result",
       "frame_index": i,
       "timestamp_ms": 1234,
       "result": {...}
     })
  ↓
6. 任务完成:
   status="completed"
   发送完成回调: {"event": "job_completed", "total_frames": N, "duration_ms": M}
  ↓
7. 异常处理:
   任务失败: status="failed", 回调: {"event": "job_failed", "error": "..."}
   任务取消: status="cancelled", 回调: {"event": "job_cancelled"}
```

## 九、关键技术方案

### 9.1 JWT认证方案

```
注册流程:
Client → POST /api/v1/auth/register (username, password)
      → AuthService 校验唯一性
      → bcrypt + 随机salt (Rounds ≥ 10) 加密密码
      → 保存User到数据库
      → 生成JWT Token (userId, username, role)
      → 返回 {token, user}

登录流程:
Client → POST /api/v1/auth/login (username, password)
      → AuthService 查询User
      → bcrypt 验证密码
      → 生成JWT Access Token (有效期2小时)
      → 生成Refresh Token (有效期7天)
      → 返回 {token, refreshToken, user}

请求鉴权:
Client → 任意API (Header: Authorization: Bearer <token>)
      → SecurityConfig JWT过滤器
      → 验证签名 + 检查过期
      → 通过: 提取userId/role到SecurityContext
      → 失败: 返回 401 {code: 1002, message: "认证失败"}
```

**JWT Token结构：**

| 字段 | Header | Payload | Signature |
|---|---|---|---|
| 示例 | `{"alg":"HS256","typ":"JWT"}` | `{"sub":"userId","username":"zhangsan","role":"USER","iat":...,"exp":...}` | HMAC-SHA256(Header.Payload, secret) |

### 9.2 文件上传与存储方案

```
上传流程:
1. 前端 FileUploader 选择文件 → 前端校验:
   - 图片: jpg/png/bmp, ≤ 10MB
   - 视频: mp4/avi/mov, ≤ 200MB

2. Axios POST (multipart/form-data) → Java后端 /api/v1/files/upload

3. Java后端 FileStorageService:
   - 校验文件MIME类型（白名单）
   - 校验文件大小（spring.servlet.multipart 配置）
   - 生成唯一文件名: {yyyyMMdd}/{uuid}.{ext}
   - 存储到本地文件系统: ./uploads/{yyyyMMdd}/{uuid}.{ext}
   - 返回可访问URL: /files/{yyyyMMdd}/{uuid}.{ext}

4. 推理请求使用 file_url 传参而非文件二进制流
```

### 9.3 WebSocket实时推送方案

```
建立连接:
前端: new WebSocket('ws://host:8080/ws/alerts?token=<JWT>')
后端: WebSocketConfig 注册 /ws/alerts 端点
     → WebSocket握手拦截器验证JWT Token
     → 通过: 建立连接，将session加入AlertSessionManager
     → 失败: 返回403关闭连接

推送流程:
Alert Agent检测到异常 → AlertService.saveAlert()
→ WebSocket推送: sessionManager.broadcast({
    type: "new_alert",
    payload: alert
  })
→ 前端 onmessage 解析 → 更新AlertStore → UI更新

心跳保活:
后端每30秒发送 {"type":"ping"}
前端回复 {"type":"pong"}
超时未回复 → 后端主动关闭 → 前端自动重连（指数退避）
```

### 9.4 异步视频任务方案

```
任务生命周期:
1. 创建任务:
   Java后端: POST /api/v1/jobs → 生成job_id(UUID) → 保存Job(status="queued")
   → 转发至算法服务: POST /api/v1/jobs (含callback_url)

2. 进度查询（前端轮询）:
   每2秒 GET /api/v1/jobs/{job_id}
   → Java后端查Job表返回 {status, progress, ...}
   → 若已收到callback的frame_result，累积更新progress

3. 实时结果推送（callback + WebSocket）:
   算法服务逐帧处理 → POST callback_url (frame_result)
   → Java后端 AlgorithmCallbackController 接收
   → 更新Job.progress + 存入DetectionResult
   → WebSocket推送帧结果到前端

4. 任务终态:
   completed: callback {"event":"job_completed"} → 更新Job.status="completed"
   failed: callback {"event":"job_failed"} → 更新Job.status="failed" + error_message
   cancelled: 前端 POST /cancel → Java后端转发算法服务 → 算法服务停止处理
```

### 9.5 前端检测结果可视化方案

**车牌检测框叠加：**

```
技术方案: HTML5 Canvas + 绝对定位
流程:
1. <img> 标签加载原图
2. 根据原图尺寸在 <canvas> 上绘制
3. 读取 result.bbox → canvas.drawRect(x1, y1, x2-x1, y2-y1, stroke="green")
4. canvas.fillText(plate_number, x1, y1-5, color="green")
5. ResultPanel 右侧列表同步展示
```

**人体骨架可视化：**

```
技术方案: Canvas + 关键点连线
流程:
1. 遍历 result.keypoints 数组
2. 绘制关键点: canvas.drawCircle(x, y, radius=3, fill="white")
3. 绘制骨架连线: 按预定义的连接关系 connectMap 连接相邻关键点
  - 头部: nose ↔ left_eye ↔ left_ear
  - 躯干: left_shoulder ↔ right_shoulder ↔ left_hip ↔ right_hip
  - 手臂: shoulder ↔ elbow ↔ wrist
  - 腿部: hip ↔ knee ↔ ankle
4. 关键点颜色按score渐变: score≥0.8 → 绿色, 0.5-0.8 → 黄色, <0.5 → 灰色
```

### 9.6 防误触发机制（车主手势）

```
判定条件（同时满足才触发）:
1. 置信度 ≥ 0.7
2. 持续时间 ≥ 0.5秒（连续N帧持续检测到同一手势）
3. 手势变化缓冲: 切换手势后需在新手势上维持0.3秒才生效

视觉反馈:
1. 识别到手势但未满足触发条件 → 控件半透明高亮（50%透明度）
2. 满足触发条件 → 控件短暂全亮200ms → 执行对应操作
3. 手势丢失 → 控件恢复常态，不执行操作
```

## 十一、安全性设计

| 安全层面 | 设计方案 | 实现方式 |
|---|---|---|
| **传输安全** | 全站HTTPS（可选TLS 1.2+） | Nginx反向代理配置SSL证书 |
| **认证安全** | JWT无状态认证 | Spring Security + jjwt库，Token有效期2小时 |
| **密码安全** | bcrypt + 随机salt | Spring Security BCryptPasswordEncoder，Rounds ≥ 10 |
| **文件上传安全** | 白名单校验（jpg/png/bmp/mp4/avi/mov） | FileStorageService校验MIME类型和后缀 |
| **文件大小限制** | 图片≤10MB，视频≤200MB | Spring Boot Multipart配置 |
| **SQL注入防护** | ORM参数化查询 | Spring Data JPA，杜绝拼接SQL |
| **XSS防护** | 前端输出转义 | Vue 3默认转义 + Content-Security-Policy头 |
| **CSRF防护** | JWT无状态（不依赖Cookie） | Token放在Authorization Header而非Cookie |
| **CORS控制** | 仅允许前端域名跨域 | CorsConfig白名单配置 |
| **API限流** | 可选（后期扩展） | Spring拦截器 + Guava RateLimiter |
| **日志脱敏** | 车牌号可选脱敏存储 | 日志写入前将车牌中间2位替换为 `***` |
