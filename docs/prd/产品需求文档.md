## 四、需求背景

### 产品 / 数据现状

随着智能驾驶与车联网技术的快速发展，车载摄像头已成为现代汽车的标配传感器。如何充分利用车载视觉能力，在**辅助驾驶安全**、**人机自然交互**、**系统智能运维**三个维度上提供可靠的技术支撑，是当前智能座舱与ADAS领域的热点问题。

当前行业存在以下技术缺口：

| 场景 | 现状与痛点 | 价值 |
|---|---|---|
| **道路车辆信息感知** | 驾驶员在复杂路况下需快速识别周围车辆信息（车牌、颜色等），人工观察效率低且易遗漏 | 辅助驾驶决策、违章举报取证、停车场自动管理 |
| **交警手势理解** | 自动驾驶系统在无信号灯路口或交通管制场景下，无法正确理解交警手势指令 | 填补自动驾驶在人工指挥场景下的感知盲区 |
| **车内手势交互** | 传统物理按键/触屏操作在驾驶过程中分散注意力，存在安全隐患 | 提供非接触式、直觉化的车内控制方式 |
| **系统智能运维** | 融合多AI模型的Web系统产生大量日志与异常事件，传统人工盯屏运维效率低下 | 引入Agent技术实现自动监控与告警，提升运维自动化水平 |

### 用户调研

| 角色 | 职责描述 | 核心需求 |

### 竞品分析

|  | 主要信息 | 关键结论 | 说明 |
|---|---|---|---|
| **百度 Apollo** | 开源自动驾驶平台，支持视觉感知（目标检测、车道线、交通标志等） | 功能全面但部署复杂，缺乏手势交互和Agent运维模块 | 参考其视觉感知流水线设计 |
| **Mobileye** | 商用ADAS视觉芯片方案，提供前装车规级感知系统 | 性能优秀但封闭生态，不支持自定义手势 | 参考其多任务视觉处理架构 |
| **MediaPipe** | Google开源框架，提供人体关键点、手部关键点等预训练模型 | 轻量高效，适合本项目快速搭建手势识别原型 | 直接采用其Hands和Pose模型 |
| **Tesla Vision** | 纯视觉自动驾驶方案，端到端神经网络 | 行业标杆但技术门槛极高，训练资源需求巨大 | 参考其视觉化UI设计理念 |

> **本项目差异化定位：** 聚焦"视觉感知 + 人机交互 + Agent运维"三位一体的Web系统，不做车规级硬件适配，而是以Web端完整的交互体验展示AI模型推理全链路。

## 六、功能详细说明

### 产品流程图

**系统整体流程：**

```
用户登录 → 选择功能模块 → 上传图片/视频/开启摄像头
    → Java后端接收请求 → 调用Python算法微服务（HTTP REST）
    → 算法推理 → 返回结构化结果 → Java后端存入数据库
    → 前端可视化展示 + WebSocket实时推送
    → Alert Agent监控日志 → 异常检测 → LLM摘要生成 → 多渠道推送
```

**车牌识别数据流（图片，同步）：**

```
用户上传图片 → Java后端保存文件并获取URL
    → POST /api/v1/inference/image (task_type: "license_plate", image_url)
    → Python算法服务：YOLOv8车牌检测 → PaddleOCR字符识别 → 颜色分类
    → 返回JSON（含plate_number、plate_color、bbox、confidence等）
    → Java后端存入数据库 → 返回结果给前端
    → 前端渲染标注图 + 结果列表
```

**交警手势识别数据流（图片）：**

```
用户上传图片 → Java后端保存文件并获取URL
    → POST /api/v1/inference/image (task_type: "police_gesture", image_url)
    → Python算法服务：MediaPipe Pose关键点检测 → 手势分类
    → 返回JSON（含gesture_code、gesture_name、keypoints、confidence等）
    → Java后端存入数据库 → 返回结果给前端
    → 前端渲染人体骨架 + 手势标签
```

**视频异步任务数据流：**

```
用户上传视频 → Java后端创建异步任务
    → POST /api/v1/jobs (task_type, video_url, callback_url, options)
    → Python算法服务排队处理 → 抽帧 → 逐帧推理
    → 通过callback_url回传实时结果 → Java后端WebSocket推送前端
    → 前端轮询 GET /api/v1/jobs/{job_id} 获取进度
    → 任务完成/失败/取消时状态更新
```

**告警智能体数据流：**

```
各模块写JSON日志 → Alert Agent轮询/监听日志流
    → 异常模式匹配 → 判定告警级别
    → 调用LLM API生成结构化告警摘要
    → 写入告警表 → WebSocket推送前端 + 邮件/Webhook通知
```

### 功能说明

#### 模块一：道路车辆车牌识别（20分，P0）

| 序号 | 功能 | 功能详细说明 |
|---|---|---|
| 1 | 图片上传识别 | 支持jpg/png/bmp格式，单张≤10MB；Java后端保存文件后传`image_url`给算法服务；输出车牌号码（plate_number）、颜色（plate_color: blue/green/white）、检测框bbox（x1,y1,x2,y2）+ 中心位置（center_x, center_y, width, height）、检测置信度+OCR置信度+综合置信度；返回标注图URL（annotated_image_url）；前端原图叠加绿色检测框+车牌号标签 |
| 2 | 视频流输入识别 | 支持mp4/avi/mov格式≤200MB；固定帧率抽帧（默认2fps）；IoU匹配关联帧间同车牌；按时间轴返回识别结果列表含帧号、时间戳、追踪ID |
| 3 | 历史记录存储查询 | 按时间范围、车牌号码（模糊匹配）、颜色筛选；分页展示每页20条；支持查看详情（标注原图+JSON结果+置信度）；可选CSV导出 |
| 4 | 场景化沙盘集成 | （可选，5分）将识别流水线部署到智慧交通管理系统场景化沙盘中验证 |
| 5 | 复杂场景鲁棒性 | （可选，3分）雨雪雾等复杂场景中正确检测并识别车牌 |

> **技术方案：** 车牌检测使用YOLOv8n/s；字符识别使用PaddleOCR PP-OCRv4；颜色分类使用颜色直方图+CNN分类器。推荐数据集：CCPD、CRPD。

**验收标准：**
- [ ] 单张图片上传后5秒内返回识别结果
- [ ] 标准光照条件下，国内蓝牌/绿牌车牌识别准确率 ≥ 85%
- [ ] 视频流能正确处理并展示逐帧识别结果
- [ ] 历史记录支持按条件查询并正确展示

#### 模块二：交警手势识别（20分，P0）

| 序号 | 功能 | 功能详细说明 |
|---|---|---|
| 1 | 8种标准手势识别 | 停止信号（STOP）、直行信号（GO_STRAIGHT）、左转弯信号（LEFT_TURN）、左转弯待转信号（LEFT_WAIT）、右转弯信号（RIGHT_TURN）、变道信号（LANE_CHANGE）、减速慢行信号（SLOW_DOWN）、靠边停车信号（PULL_OVER）。输出gesture_code（英文编码）+ gesture_name（中文名称） |
| 2 | 人体关键点检测 | MediaPipe Pose提取人体关键点；每条关键点含name（如left_shoulder, left_wrist）+ x,y坐标 + score置信度；单帧检测≤100ms |
| 3 | 时序动作分类 | 连续N帧（建议30帧@30fps=1秒）关键点坐标序列输入ST-GCN/LSTM分类器；完整分类≤500ms |
| 4 | Web端实时展示 | 视频播放区+人体骨架可视化（按keypoints数组绘制白色骨架连线）；右上角手势标签（大号字体+颜色标签）+ confidence置信度；手势历史时间线 |
| 5 | 复杂场景鲁棒性 | （可选，3分）多行人场景定位主要人物；不同光照条件检测；部分遮挡场景正确分类 |

> **技术方案：** 人体关键点检测使用MediaPipe Pose（BlazePose）；时序动作分类使用ST-GCN或LSTM+FC。推荐数据集：CTPGR（交警手势数据集）。

**验收标准：**
- [ ] 8种交警手势均能正确识别，单种手势Top-1准确率 ≥ 80%
- [ ] 视频流处理延迟 ≤ 2秒（含检测+分类+展示）
- [ ] Web端正确显示人体骨架+手势标签

#### 模块三：车主手势控车（20分，P0）

| 序号 | 功能 | 功能详细说明 |
|---|---|---|
| 1 | 6种预定义手势识别 | 🖐手掌张开（gesture_code: 001）→启动/唤醒；✊握拳（002）→确认/执行；☝️单指画圈（003）→调节音量；👈👉左右滑动（004）→切换功能；👍👎拇指上下（005/006）→接听/挂断；👋挥手（007）→返回主页。输出gesture_code（数字编码）+ confidence置信度 + keypoints数组（每个关键点含name, x, y, score） |
| 2 | 手部关键点检测 | MediaPipe Hands提取手部关键点（wrist等），每条关键点含name + x,y坐标 + score置信度；静态手势用几何规则判定，动态手势由后端连续发送识别请求，通过时序运动特征判定 |
| 3 | 模拟车辆控制面板 | 音量滑块、空调温度控制、电话接听/挂断按钮、功能标签页切换（音乐/导航/电话/空调）、歌曲切换、返回主页 |
| 4 | 防误触发机制 | 持续时间阈值≥0.5秒；置信度门槛≥0.7；视觉确认反馈（控件短暂高亮200ms） |
| 5 | 手势自定义扩展 | （可选，5分）手势数据采集工具+MediaPipe Model Maker微调训练+自定义手势绑定控制操作 |

> **技术方案：** 手部关键点检测使用MediaPipe Hands；手势分类使用几何规则+轻量MLP/CNN。推荐数据集：HaGRID、NVGesture。

**验收标准：**
- [ ] 6种手势均能正确识别并触发对应控制操作
- [ ] 单次手势识别延迟 ≤ 500ms
- [ ] 控制面板各控件手势交互正常
- [ ] 连续非手势动作（如挠头、整理头发）不会误触发

| 1 | 监控日志体系 | 车牌识别日志、交警手势识别日志、车主手势识别日志、用户操作日志、系统运行日志（QPS/响应时间/CPU/内存/LLM Token消耗）；统一JSON格式含timestamp/level/module/event/detail/trace_id |
| 2 | Alert Agent感知 | 8种异常事件检测：车牌识别连续失败（1分钟失败率>30%）、车牌识别全面故障（3分钟成功率<10%）、手势置信度持续偏低（5分钟均值<0.5）、LLM API超时（>30秒或1分钟≥3次）、Token超额（80%/100%阈值）、未授权访问、数据库连接失败、CPU/内存过高 |
| 3 | Alert Agent决策 | 三级告警：💡提示（蓝）、⚠️警告（黄）、🔴严重（红）；调用LLM API生成结构化告警摘要（含alert_id、severity、title、summary、anomaly_type、occurred_at、affected_module、metrics、suggested_actions） |
| 4 | Alert Agent执行 | Web端WebSocket实时推送+通知铃铛；邮件通知（SMTP）；可选Webhook推送（飞书/钉钉/企业微信） |
| 5 | 告警可视化 | 告警时间线（按时间倒序，含级别标签+处理状态）；告警详情页（LLM摘要+原始日志+指标图表）；统计仪表盘（今日告警总数+级别饼图+7天趋势折线图+模块柱状图）；严重告警回放（前后5分钟日志时间线） |

> **技术方案：** Agent框架可选用LangChain或简化规则引擎+LLM API；LLM API选择DeepSeek-V3或通义千问；实时推送使用WebSocket（FastAPI WebSocket）。

**验收标准：**
- [ ] 4类监控日志均正常记录并可查询
- [ ] Agent能检测到至少5种异常事件并正确分级
- [ ] LLM生成的告警摘要包含所要求的全部结构化字段
- [ ] 告警通知能通过Web端实时推送 + 至少1种外部渠道发送
- [ ] 告警仪表盘正确展示统计数据

**Java后端 ↔ Python算法微服务（内部接口，不直接暴露给前端）：**

| 方法 | 路径 | 描述 |
|---|---|---|
| POST | `/api/v1/inference/image` | 统一图片推理（`task_type` + `image_url`，同步返回结果） |
| POST | `/api/v1/jobs` | 创建视频异步任务（含`callback_url` + `options`） |
