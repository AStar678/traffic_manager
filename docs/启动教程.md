# VisionDrive 启动教程

本文档说明如何在本机启动 VisionDrive 的四个服务：

| 服务 | 技术栈 | 默认地址 |
|---|---|---|
| 前端 | Vue 3 + Vite | http://127.0.0.1:5173 |
| 后端 | Java Spring Boot | http://127.0.0.1:8080 |
| 算法服务 | Python FastAPI | http://127.0.0.1:8000 |
| 虚拟摄像头服务 | Python FastAPI | http://127.0.0.1:8010 |

## 1. 环境要求

请先确认本机已安装以下环境：

- Node.js 和 npm
- Java 17 或更高版本
- Maven
- Python 3.12

可以用以下命令检查：

```bash
node --version
npm --version
java --version
mvn --version
python3 --version
```

## 2. 首次安装依赖

进入项目根目录：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager
```

安装前端依赖：

```bash
cd frontend
npm install
cd ..
```

创建并安装算法服务 Python 环境：

```bash
cd algorithm
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
cd ..
```

摄像头服务复用算法服务的 Python 虚拟环境。如果后续摄像头服务依赖有新增，可以执行：

```bash
algorithm/.venv/bin/pip install -r camera_service/requirements.txt
```

后端依赖由 Maven 在启动或构建时自动下载。

## 3. 前台启动

开发调试时建议分别打开 4 个终端窗口，按下面顺序启动。

### 3.1 启动算法服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/algorithm
ALGORITHM_HOST=127.0.0.1 .venv/bin/python main.py
```

启动成功后可以访问：

```bash
curl http://127.0.0.1:8000/health
```

### 3.2 启动虚拟摄像头服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/camera_service
../algorithm/.venv/bin/python -m uvicorn main:app --host 127.0.0.1 --port 8010
```

启动成功后可以访问：

```bash
curl http://127.0.0.1:8010/health
```

摄像头服务管理台地址：

```text
http://127.0.0.1:8010/ui
```

### 3.3 启动后端服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/backend
mvn spring-boot:run
```

如果配置了本地邮件告警环境文件 `backend/.env.local`，使用下面的方式启动后端：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/backend
set -a
source .env.local
set +a
mvn spring-boot:run
```

启动成功后可以访问：

```bash
curl http://127.0.0.1:8080/api/v1/alerts
```

### 3.4 启动前端服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/frontend
npm run dev -- --host 127.0.0.1
```

浏览器打开：

```text
http://127.0.0.1:5173
```

## 4. 后台启动

如果希望服务在后台持续运行，可以使用 macOS 自带的 `screen`。

进入项目根目录：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager
```

启动四个后台会话：

```bash
screen -dmS visiondrive-algorithm env ALGORITHM_HOST=127.0.0.1 algorithm/.venv/bin/python algorithm/main.py
screen -dmS visiondrive-camera algorithm/.venv/bin/python -m uvicorn main:app --app-dir camera_service --host 127.0.0.1 --port 8010
screen -dmS visiondrive-backend bash -lc 'cd backend && mvn spring-boot:run'
screen -dmS visiondrive-frontend bash -lc 'cd frontend && npm run dev -- --host 127.0.0.1'
```

查看后台会话：

```bash
screen -ls
```

进入某个会话查看实时日志：

```bash
screen -r visiondrive-backend
```

在会话中按 `Ctrl+A`，再按 `D` 可以退出但不停止服务。

## 5. 健康检查

服务启动完成后，执行：

```bash
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8010/health
curl -o /tmp/visiondrive-frontend-check.html -w "%{http_code}\n" http://127.0.0.1:5173/
curl -o /tmp/visiondrive-backend-check.txt -w "%{http_code}\n" http://127.0.0.1:8080/api/v1/alerts
```

正常结果：

- 算法服务返回 `status: ok`
- 摄像头服务返回 `status: ok`
- 前端返回 HTTP `200`
- 后端返回 HTTP `200`

也可以检查端口监听：

```bash
lsof -nP -iTCP:5173 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:8000 -sTCP:LISTEN
lsof -nP -iTCP:8010 -sTCP:LISTEN
```

## 6. 停止和重启

如果是前台启动，在对应终端按 `Ctrl+C` 停止。

如果是 `screen` 后台启动，可以逐个停止：

```bash
screen -S visiondrive-frontend -X quit
screen -S visiondrive-backend -X quit
screen -S visiondrive-camera -X quit
screen -S visiondrive-algorithm -X quit
```

确认端口释放：

```bash
lsof -nP -iTCP:5173 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:8000 -sTCP:LISTEN
lsof -nP -iTCP:8010 -sTCP:LISTEN
```

如果没有输出，表示端口已经释放。随后按第 3 节或第 4 节重新启动即可。

## 7. 常见问题

### 端口已被占用

先查占用进程：

```bash
lsof -nP -iTCP:端口号 -sTCP:LISTEN
```

例如：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
```

确认可以停止后再结束对应进程：

```bash
kill 进程号
```

### 算法服务无法访问模型文件

检查 `algorithm/config.py` 中的模型路径配置，或通过环境变量覆盖：

```bash
LICENSE_PLATE_SOURCE_DIR=/实际路径 ALGORITHM_HOST=127.0.0.1 .venv/bin/python main.py
```

### 前端摄像头接口地址不对

前端默认读取：

```text
http://127.0.0.1:8010
```

如需修改，可以设置 `VITE_CAMERA_URL` 后再启动前端：

```bash
VITE_CAMERA_URL=http://127.0.0.1:8010 npm run dev -- --host 127.0.0.1
```
