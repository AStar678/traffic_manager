# VisionDrive 启动教程

本文档说明如何在本机启动 VisionDrive 的五个服务：

| 服务 | 技术栈 | 默认地址 |
|---|---|---|
| 前端 | Vue 3 + Vite | http://127.0.0.1:5173 |
| 后端 | Java Spring Boot | http://127.0.0.1:8080 |
| 车牌识别服务 | Python FastAPI | http://127.0.0.1:8000 |
| 交警手势服务 | Python FastAPI | http://127.0.0.1:8001 |
| 车主手势服务 | Python FastAPI | http://127.0.0.1:8002 |

## 1. 环境要求

请先确认本机已安装以下环境：

- Node.js 和 npm
- Java 17 或更高版本
- Maven
- Python 3.12
- FFmpeg（主服务从视频、RTSP、沙盘或本机设备取帧）

可以用以下命令检查：

```bash
node --version
npm --version
java --version
mvn --version
python3 --version
ffmpeg -version
```

## 2. 首次安装依赖

进入项目根目录：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager
```

安装前端依赖：

```bash
cd frontend
npm install
cd ..
```

为三个算法服务分别创建并安装 Python 环境：

```bash
cd algorithm
python3 -m venv license/.venv
license/.venv/bin/pip install -r license/requirements.txt
python3 -m venv police/.venv
police/.venv/bin/pip install -r police/requirements.txt
python3 -m venv gesture/.venv
gesture/.venv/bin/pip install -r gesture/requirements.txt
cd ..
```

后端依赖由 Maven 在启动或构建时自动下载。

## 3. 前台启动

开发调试时建议分别打开终端窗口，按下面顺序启动。

### 3.1 启动三个算法服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/algorithm
LICENSE_ALGORITHM_HOST=127.0.0.1 license/.venv/bin/python -m license.main
POLICE_ALGORITHM_HOST=127.0.0.1 police/.venv/bin/python -m police.main
GESTURE_ALGORITHM_HOST=127.0.0.1 gesture/.venv/bin/python -m gesture.main
```
```bash
windows:
终端 1 — 车牌识别：
$env:LICENSE_ALGORITHM_HOST = "127.0.0.1"; .\license\.venv\Scripts\python -m license.main

终端 2 — 交警手势：
$env:POLICE_ALGORITHM_HOST = "127.0.0.1"; .\police\.venv\Scripts\python -m police.main

终端 3 — 车主手势：
$env:GESTURE_ALGORITHM_HOST = "127.0.0.1"; .\gesture\.venv\Scripts\python -m gesture.main
```
以上三条命令需要在三个终端中分别运行。默认端口依次为 8000、8001、8002。

启动成功后可以访问：

```bash
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8001/health
curl http://127.0.0.1:8002/health
```

交警服务健康检查中的 `models.policeGesture.modelExists` 应为 `true`。如交警识别工程不在默认的学期实训目录，可在启动前设置：

```bash
POLICE_GESTURE_SOURCE_DIR=/实际路径/交警指令识别 POLICE_ALGORITHM_HOST=127.0.0.1 police/.venv/bin/python -m police.main
```

车牌和交警服务默认各使用 2 个工作进程并行推理，可通过 `LICENSE_ALGORITHM_WORKERS`、`POLICE_ALGORITHM_WORKERS` 调整。车主手势为了保持录入状态一致固定为单进程，使用 `GESTURE_ALGORITHM_THREADS` 控制只读识别线程并发（默认 40）。

### 3.2 启动后端服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/backend
mvn spring-boot:run
```

如果配置了本地邮件告警环境文件 `backend/.env.local`，使用下面的方式启动后端：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/backend
set -a
source .env.local
set +a
mvn spring-boot:run
```

启动成功后可以访问：

```bash
curl http://127.0.0.1:8080/api/v1/alerts
curl http://127.0.0.1:8080/api/v1/cameras/slots
```

主服务会自动创建 3 个摄像头槽位。帧文件默认位于 `backend/uploads/camera-frames`，车牌与交警算法通过本地文件路径读取，不再启动独立摄像头服务，也不再使用 8010、WebRTC 或图片网络中转。

### 3.3 启动前端服务

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager/frontend
npm run dev -- --host 127.0.0.1
```

浏览器打开：

```text
http://127.0.0.1:5173
```

## 4. 后台启动

如果希望服务在后台持续运行，可以使用 macOS 自带的 `screen`。

进入项目根目录：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager
```

启动五个后台会话：

```bash
screen -dmS visiondrive-license bash -lc 'cd algorithm && LICENSE_ALGORITHM_HOST=127.0.0.1 license/.venv/bin/python -m license.main'
screen -dmS visiondrive-police bash -lc 'cd algorithm && POLICE_ALGORITHM_HOST=127.0.0.1 police/.venv/bin/python -m police.main'
screen -dmS visiondrive-gesture bash -lc 'cd algorithm && GESTURE_ALGORITHM_HOST=127.0.0.1 gesture/.venv/bin/python -m gesture.main'
screen -dmS visiondrive-backend bash -lc 'cd backend && mvn spring-boot:run'
screen -dmS visiondrive-frontend bash -lc 'cd frontend && npm run dev -- --host 127.0.0.1'
```

查看后台会话：

```bash
screen -ls
```

进入某个会话查看实时日志：

```bash
screen -r visiondrive-backend
```

在会话中按 `Ctrl+A`，再按 `D` 可以退出但不停止服务。

## 5. 健康检查

服务启动完成后，执行：

```bash
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8001/health
curl http://127.0.0.1:8002/health
curl -o /tmp/visiondrive-frontend-check.html -w "%{http_code}\n" http://127.0.0.1:5173/
curl -o /tmp/visiondrive-backend-check.txt -w "%{http_code}\n" http://127.0.0.1:8080/api/v1/alerts
curl -o /tmp/visiondrive-camera-slots.json -w "%{http_code}\n" http://127.0.0.1:8080/api/v1/cameras/slots
```

正常结果：

- 三个算法服务均返回 `status: ok`
- 主服务摄像头槽位接口返回 3 个槽位
- 前端返回 HTTP `200`
- 后端返回 HTTP `200`

也可以检查端口监听：

```bash
lsof -nP -iTCP:5173 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:8000 -sTCP:LISTEN
lsof -nP -iTCP:8001 -sTCP:LISTEN
lsof -nP -iTCP:8002 -sTCP:LISTEN
```

## 6. 停止和重启

如果是前台启动，在对应终端按 `Ctrl+C` 停止。

如果是 `screen` 后台启动，可以逐个停止：

```bash
screen -S visiondrive-frontend -X quit
screen -S visiondrive-backend -X quit
screen -S visiondrive-gesture -X quit
screen -S visiondrive-police -X quit
screen -S visiondrive-license -X quit
```

确认端口释放：

```bash
lsof -nP -iTCP:5173 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:8000 -sTCP:LISTEN
lsof -nP -iTCP:8001 -sTCP:LISTEN
lsof -nP -iTCP:8002 -sTCP:LISTEN
```

如果没有输出，表示端口已经释放。随后按第 3 节或第 4 节重新启动即可。

## 7. 常见问题

### 端口已被占用

先查占用进程：

```bash
lsof -nP -iTCP:端口号 -sTCP:LISTEN
```

例如：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
```

确认可以停止后再结束对应进程：

```bash
kill 进程号
```

### 算法服务无法访问模型文件

检查各服务的 `config.py`，或通过环境变量覆盖：

```bash
cd algorithm
LICENSE_PLATE_SOURCE_DIR=/实际路径 LICENSE_ALGORITHM_HOST=127.0.0.1 license/.venv/bin/python -m license.main
```

### 摄像头没有画面

先确认 FFmpeg 可用，再检查主服务槽位和单帧接口：

```bash
ffmpeg -version
curl http://127.0.0.1:8080/api/v1/cameras/slots
curl -o /tmp/camera-1.jpg http://127.0.0.1:8080/api/v1/cameras/slots/1/frame.jpg
```

前端“摄像头”模块可以独立设置每个槽位的输入类型、名称、沙盘机位、流地址、设备索引或上传文件。Docker 部署时后端与车牌/交警算法必须挂载同一个 `CAMERA_FRAME_DIR` 共享卷。
