## 服务分布

| 服务 | 监听地址 | 说明 |
|---|---|---|
| Java 主服务 | `0.0.0.0:8080` | 用户、数据库、三路摄像头和并发编排 |
| 车牌算法 | `127.0.0.1:8000` | CUDA/RTX 3080，仅供主服务调用 |
| 交警算法 | `127.0.0.1:8001` | CUDA/RTX 3080，仅供主服务调用 |
| 车主手势 | `0.0.0.0:8002` | 本机前端的 WebSocket/管理接口 |
| WebRTC 信令 | `0.0.0.0:8003` | 三路视频信令、短期 TURN 凭证、持续帧发布 |
| coturn | `100.73.64.55:3478` | WebRTC TCP/UDP 中继；UDP 端口范围 `49160-49200` |
| 本机前端 | `127.0.0.1:5173` | 通过 Vite 代理连接服务器 |

浏览器视频使用 WebRTC，默认强制走 TURN 中继以避免局域网/Tailscale 链路抖动。WebRTC 服务使用三个独立 FFmpeg 子进程直接读取服务器文件或 RTSP，不通过 HTTP 传输源视频；服务启动后即持续发布三路最新帧到 `/home/aoxiang/visiondrive/shared/camera-frames`。主服务、车牌算法和交警算法只通过该目录传递本地文件路径，页面是否打开不影响识别。

`/home/aoxiang/visiondrive/config/turn.env` 保存临时 TURN 凭证签名密钥，权限为 `600`；coturn 的同一密钥位于 `/etc/turnserver.conf`，文件必须为 `root:turnserver 640`。密钥不得提交到 Git。前端只获取一小时有效的 HMAC 凭证，不接触共享密钥。

## 后续修改的固定流程

1. 只在本机工作区修改代码。
2. 本机完成前端构建、后端测试与算法测试。
3. 运行同步脚本；脚本只在本机测试全部通过后上传并重启服务器。
4. 自动检查五个应用服务、coturn TCP 端口和 GPU 进程。

日常发布：

```bash
cd /Users/aoxiang/Desktop/软件工程学期实训/code/traffic_manager
./scripts/deploy-server.sh
```

首次部署或模型发生变化：

```bash
SYNC_MODELS=1 ./scripts/deploy-server.sh
```

仅检查服务器：

```bash
./scripts/check-server.sh
```

`SKIP_TESTS=1` 只用于已明确完成同一版本本机测试的紧急重复部署，常规发布禁止使用。

## 交警识别回归基线

测试集位于本机 `/Users/aoxiang/Desktop/软件工程学期实训/交警指令识别/test`。评估只需将 10 个 CSV 同步到服务器 `/home/aoxiang/visiondrive/test/police`，骨架坐标直接使用模型目录的 `generated/coords/test`，避免重复上传约 978 MB 视频。

2026-07-12 基线为 10 个样本、61,581 帧：本机 CPU 准确率 `0.464478`，服务器 CUDA 准确率 `0.464348`，仅 8 帧差异（99.987% 一致）。姿态权重 SHA-256 为 `ce2931a...c52a0e`，LSTM 权重为 `cbc6b8a...ca2cb`。这说明服务器与本机模型、预处理及时序输出一致；较低的逐帧绝对准确率属于原始模型在该标注集上的基线，后续提升需要重新训练或校准模型，而不是网络传输修复。

云 MySQL 的账号限制不允许 GPU 服务器公网地址直接登录，因此主服务继续通过 SSH 隧道访问数据库。GPU 服务器使用独立的 `~/.ssh/visiondrive_db` 密钥，数据库账号、库名、JWT 和已有数据均沿用 `backend/.env.local`，不会写入仓库，也不会复制本机已有私钥。

## 本机连接云数据库

云数据库不直接暴露给本机，必须先建立 SSH 隧道：

```bash
./scripts/db-tunnel.sh start   # 后台启动隧道
./scripts/db-tunnel.sh status  # 检查本机 13306 端口
./scripts/db-tunnel.sh test    # 执行数据库只读握手测试
./scripts/db-tunnel.sh stop    # 不再使用时关闭隧道
```

数据库客户端连接地址固定为 `127.0.0.1:13306`，库名、用户名和密码读取 `backend/.env.local`。`run-backend-cloud.sh` 会自动复用已经运行的隧道，未运行时仍会临时创建并在后端退出时清理。
