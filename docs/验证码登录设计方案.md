## 一、需求概述

为 VisionDrive 系统增加**手机号验证码登录**和**邮箱验证码登录**两种免密登录方式，替代或补充传统的账号密码登录。用户无需记忆密码，通过获取一次性验证码即可完成身份认证。

### 1.1 核心场景

| 场景 | 说明 |
|---|---|
| 手机号 + 短信验证码登录 | 用户输入手机号，获取短信验证码，验证通过后登录 |
| 邮箱 + 邮件验证码登录 | 用户输入邮箱地址，获取邮件验证码，验证通过后登录 |
| 首次登录自动注册 | 未注册的手机号/邮箱首次登录时自动创建账号 |
| Token 续期 | 登录成功后返回 JWT Access Token + Refresh Token |

### 1.2 非功能性需求

- 验证码有效期：**5分钟**
- 发送频率限制：**60秒/次**（同手机号/邮箱）
- 每日发送上限：**10次/天**（同手机号/邮箱，防刷）
- 验证码错误次数上限：**5次**（同验证码，超过则失效）
- Access Token 有效期：**2小时**（沿用现有配置）
- Refresh Token 有效期：**7天**（沿用现有配置）
- 验证码长度：**6位数字**

## 三、数据库设计

### 3.1 User 表 `t_user`

```sql
CREATE TABLE t_user (
    id              BIGINT          AUTO_INCREMENT PRIMARY KEY,
    phone           VARCHAR(20)     DEFAULT NULL COMMENT '手机号(加密存储)',
    phone_encrypted VARCHAR(255)    DEFAULT NULL COMMENT '手机号加密密文',
    email           VARCHAR(100)    DEFAULT NULL COMMENT '邮箱(加密存储)',
    email_encrypted VARCHAR(255)    DEFAULT NULL COMMENT '邮箱加密密文',
    nickname        VARCHAR(50)     DEFAULT NULL COMMENT '昵称',
    avatar_url      VARCHAR(500)    DEFAULT NULL COMMENT '头像URL',
    role            VARCHAR(20)     NOT NULL DEFAULT 'USER' COMMENT '角色: USER/ADMIN',
    status          TINYINT         NOT NULL DEFAULT 1 COMMENT '状态: 1=正常, 0=禁用',
    
    -- 审计字段
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    last_login_at   DATETIME        DEFAULT NULL COMMENT '最后登录时间',
    
    -- 索引
    UNIQUE INDEX idx_phone (phone),
    UNIQUE INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 3.2 Refresh Token 表 `t_refresh_token`

```sql
CREATE TABLE t_refresh_token (
    id              BIGINT          AUTO_INCREMENT PRIMARY KEY,
    user_id         BIGINT          NOT NULL COMMENT '用户ID',
    token           VARCHAR(500)    NOT NULL COMMENT 'Refresh Token',
    expires_at      DATETIME        NOT NULL COMMENT '过期时间',
    revoked         TINYINT         NOT NULL DEFAULT 0 COMMENT '是否已撤销: 1=是, 0=否',
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_user_id (user_id),
    UNIQUE INDEX idx_token (token)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Refresh Token 表';
```

### 3.3 验证码存储（Redis）

验证码不持久化到数据库，使用 Redis 存储，结构如下：

```
Key:   code:login:{type}:{target}
Value: {"code":"123456","attempts":0}
TTL:   300 (5分钟)

# 示例
Key:   code:login:phone:13800138000
Value: {"code":"483921","attempts":0}
TTL:   300

Key:   code:login:email:user@example.com
Value: {"code":"719254","attempts":0}
TTL:   300
```

**发送频率限制 Key**：
```
Key:   rate:send:{type}:{target}
Value: 发送次数
TTL:   60 (60秒)

Key:   rate:daily:{type}:{target}:{date}
Value: 当日发送次数
TTL:   86400 (24小时)
```

## 四、API 接口设计

### 4.1 发送验证码

```
POST /api/v1/auth/send-code
```

**请求参数**：
```json
{
  "type": "phone",          // 必填，枚举值: "phone" | "email"
  "target": "13800138000",  // 必填，手机号或邮箱地址
  "captchaToken": "xxx"     // 可选，图形验证码Token（防机器刷接口）
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "验证码已发送",
  "data": {
    "retryAfter": 60        // 下次可重新发送的秒数
  }
}
```

**错误响应**：
```json
{
  "code": 429,
  "message": "发送过于频繁，请 45 秒后再试",
  "data": null
}
```

### 4.2 验证码登录

```
POST /api/v1/auth/login/code
```

**请求参数**：
```json
{
  "type": "phone",          // 必填，枚举值: "phone" | "email"
  "target": "13800138000",  // 必填，手机号或邮箱地址
  "code": "483921"          // 必填，6位验证码
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJl...",
    "expiresIn": 7200,
    "user": {
      "id": 1,
      "phone": "138****8000",
      "email": null,
      "nickname": "用户138000",
      "avatarUrl": null,
      "role": "USER"
    }
  }
}
```

### 4.3 Token 刷新

```
POST /api/v1/auth/refresh
```

**请求参数**：
```json
{
  "refreshToken": "dGhpcyBpcyBhIHJlZnJl..."
}
```

**响应**：同登录成功，返回新的 Access Token + Refresh Token 对。

### 4.4 登出

```
POST /api/v1/auth/logout
Authorization: Bearer {accessToken}
```

**请求参数**（可选）：
```json
{
  "refreshToken": "dGhpcyBpcyBhIHJlZnJl..."
}
```

### 4.5 获取当前用户信息

```
GET /api/v1/auth/me
Authorization: Bearer {accessToken}
```

### 5.2 核心类设计

#### 5.2.1 `SecurityConfig.java` — Spring Security 配置

```text
职责：
1. 放行白名单: /auth/send-code, /auth/login/code, /auth/refresh, /swagger-ui/**, /v3/api-docs/**
2. 注册 JwtAuthenticationFilter 到 UsernamePasswordAuthenticationFilter 之前
3. 配置 CORS
4. 配置 Session 为 STATELESS（无状态）
5. 禁用 CSRF
6. 配置全局异常处理入口
```

#### 5.2.2 `JwtTokenProvider.java` — JWT 工具类

```text
核心方法:
- generateAccessToken(Long userId, String role): 生成 Access Token (2h)
- generateRefreshToken(Long userId): 生成 Refresh Token (7d)
- validateToken(String token): 验证 Token 有效性
- getUserIdFromToken(String token): 从 Token 中提取用户ID
- getRoleFromToken(String token): 从 Token 中提取用户角色

JWT Payload:
{
  "sub": "1",                 // 用户ID
  "role": "USER",             // 用户角色
  "type": "ACCESS",           // Token类型: ACCESS/REFRESH
  "iat": 1625653200,          // 签发时间
  "exp": 1625660400           // 过期时间
}
```

#### 5.2.3 `JwtAuthenticationFilter.java` — JWT 过滤器

```text
职责：
1. 从请求头 Authorization 提取 Bearer Token
2. 调用 JwtTokenProvider.validateToken() 验证
3. 验证通过后从数据库加载用户信息
4. 构造 UsernamePasswordAuthenticationToken 设置到 SecurityContext
5. 验证失败返回 401 Unauthorized
```

#### 5.2.4 `VerificationCodeService.java` — 验证码服务

```text
核心方法:
- sendCode(LoginType type, String target): 发送验证码
  → 检查频率限制 (60s + 每日10次)
  → 生成6位随机数字
  → 存储到 Redis (TTL 5min)
  → 调用对应的发送器 (SmsCodeSender / EmailCodeSender)

- verifyCode(LoginType type, String target, String code): 验证验证码
  → 从 Redis 查询验证码
  → 比对验证码
  → 检查错误次数（超5次则删除验证码，要求重新发送）
  → 验证成功删除验证码（一次性使用）

- canSend(LoginType type, String target): boolean
  → 检查是否满足发送频率要求
```

#### 5.2.5 `AuthServiceImpl.java` — 认证服务

```text
核心方法:
- loginByCode(CodeLoginRequest request): LoginResponse
  → 调用 VerificationCodeService.verifyCode() 验证验证码
  → 查询用户 (phone 或 email)
  → 如果用户不存在 → 自动注册新用户
  → 生成 Access Token + Refresh Token
  → 保存 Refresh Token 到数据库
  → 更新 lastLoginAt
  → 返回 LoginResponse

- refreshToken(String refreshToken): LoginResponse
  → 从数据库查询 Refresh Token
  → 验证是否过期 / 是否已撤销
  → 撤销旧的 Refresh Token
  → 生成新的 Access Token + Refresh Token
  → 保存新的 Refresh Token
  → 返回 LoginResponse

- logout(String accessToken, String refreshToken):
  → 撤销指定的 Refresh Token
  → 可选：将 Access Token 加入黑名单（Redis, TTL=剩余有效期）

- getCurrentUser(Long userId): UserDTO
```

#### 5.2.6 `SmsCodeSender.java` — 短信发送

```text
- 封装阿里云短信服务 SDK（或腾讯云短信 SDK）

- 配置项:
```

#### 6.2.2 `LoginForm.vue` — 登录表单核心逻辑

```text
状态管理：
- loginType: 'phone' | 'email'             — 当前选中的登录方式
- target: ''                               — 手机号或邮箱
- code: ''                                 — 6位验证码
- sendButtonText: '发送验证码' | 'Xs后重试' — 按钮文本
- sendButtonDisabled: boolean              — 按钮是否禁用
- countdown: number                        — 倒计时秒数

校验规则：
- 手机号: /^1[3-9]\d{9}$/
- 邮箱:   标准邮箱正则
- 验证码: /^\d{6}$/

交互流程：
1. 用户切换 Tab → 清空输入 + 重置倒计时
2. 用户输入手机号/邮箱 → 实时格式校验 → 校验通过后「发送验证码」按钮可用
3. 点击「发送验证码」→ 调用 POST /auth/send-code → 
   → 成功: 开始60秒倒计时，按钮变灰
   → 失败: 显示错误提示，不锁按钮
4. 用户输入6位验证码 → 实时校验
5. 点击「登录」→ 调用 POST /auth/login/code →
   → 成功: 存储 token → Pinia → localStorage → 跳转首页
```

#### 6.2.5 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  
  if (to.path === '/login') {
    // 已登录用户访问登录页 → 重定向到首页
    if (authStore.isAuthenticated) return next('/')
    return next()
  }
  
  // 未登录用户访问需要认证的页面 → 重定向到登录页
  if (!authStore.isAuthenticated) {
    return next({ path: '/login', query: { redirect: to.fullPath } })
  }
  
  next()
})
```

## 七、安全设计

### 7.1 防护措施

| 风险 | 防护措施 |
|---|---|
| 验证码暴力破解 | 5次错误后验证码失效，需重新发送；6位数字共100万组合 |
| 短信轰炸（刷接口） | 60秒/次限制 + 每日10次上限 + 可选图形验证码 |
| Token 泄露 | Access Token 短有效期(2h) + Refresh Token 可撤销 |
| 手机号/邮箱泄露 | Jasypt 加密存储，日志中脱敏展示 `138****8000` |
| 中间人攻击 | HTTPS 传输（生产环境强制） |
| CSRF | Session 无状态 + 禁用 CSRF |
| XSS | 前端输出转义 + Content-Security-Policy 头 |

### 7.2 手机号/邮箱脱敏规则

```java
// 手机号: 13800138000 → 138****8000
phone.replaceAll("(\\d{3})\\d{4}(\\d{4})", "$1****$2");

// 邮箱: testuser@example.com → t****r@example.com
email.replaceAll("(.).*(.@)", "$1****$2");
```

### 7.3 环境变量管理

所有敏感配置通过环境变量注入，严禁硬编码：

| 变量名 | 说明 |
|---|---|
| `JWT_SECRET` | JWT 签名密钥（至少256位） |
| `REDIS_PASSWORD` | Redis 密码 |
| `MAIL_PASSWORD` | 邮件服务密码 |
| `ALIYUN_ACCESS_KEY_ID` | 阿里云 AccessKey |
| `ALIYUN_ACCESS_KEY_SECRET` | 阿里云 AccessSecret |
| `JASYPT_ENCRYPTOR_PASSWORD` | 数据库字段加密密钥 |

## 十、风险评估与注意事项

| 风险 | 等级 | 应对措施 |
|---|---|---|
| 短信服务未开通 | 🔴 高 | 优先实现 Mock 模式，不影响开发联调 |
| Redis 不可用 | 🟡 中 | 提供内存缓存降级方案 |
| 手机号/邮箱隐私合规 | 🟡 中 | 加密存储 + 脱敏展示 + 隐私协议勾选 |
| Refresh Token 旋转竞争 | 🟢 低 | 数据库唯一约束 + 乐观锁 |
| 前端 Token 刷新并发冲突 | 🟢 低 | 队列化刷新请求，同一时间只发一个刷新请求 |

### 注意事项

1. **不要移除 `SecurityAutoConfiguration` 的排除**，而是移除 `exclude` 配置，启用 Security 的完整自动配置
2. **Secret Key 长度**：HS256 要求密钥至少 256 bits（32字符），生产环境务必使用强随机密钥
3. **日志安全**：验证码不应记录到日志中；手机号/邮箱在日志中应脱敏
4. **HTTPS**：生产环境必须强制 HTTPS，防止验证码和 Token 在传输中被窃取
5. **图形验证码**：建议在「发送验证码」之前增加图形验证码（滑块/点选），防止机器恶意刷接口。第一期可先不加，后续迭代增加
6. **数据合规**：手机号/邮箱属于个人敏感信息，需注意数据安全法和个人信息保护法的合规要求
